ESP Library Management — Full Project Details & Setup

Last updated: 2026-03-19

Stack ESP32 + RFID + IR MERN-style Web UI

This project combines an ESP32 hardware workflow (RFID issue/exit detection + IR people counting) with a React frontend and an Express + MongoDB backend.

1) Repository Overview

2) System Architecture

Hardware Layer (ESP32)

  • Reads RFID tag UIDs at entry and exit points.
  • Maps UID to known student IDs and book IDs.
  • Tracks people count using entry/exit IR sensors.
  • Sends HTTPS requests to backend endpoints.

Backend Layer (Express + MongoDB)

  • Receives issue/verify/count events from ESP32.
  • Stores transactions in MongoDB.
  • Exposes stats and login-based data retrieval for frontend.
  • Handles duplicate issue popup workflow.

Frontend Layer (React + Vite)

  • Displays stats: issued books, available books, people count.
  • Provides login and user dashboard views.
  • Polls duplicate popup endpoint for warning notifications.
  • Uses Vite proxy in local dev and rewrites in deployment.

3) Backend Setup

3.1 Prerequisites

  • Node.js 18+ (recommended: latest LTS).
  • A MongoDB Atlas/database URI.

3.2 Install & Run

cd UI/backend
npm install
npm run dev   # nodemon server.js
# or
npm start     # node server.js

3.3 Required Environment Variables (.env)

PORT=5000
MONGO_URI=your_mongodb_connection_string
FRONTEND_URL=http://localhost:5173
TOTAL_BOOKS=6

3.4 CORS Rules (from server.js)

  • Allowed localhost origins include 3000, 5173, and 5000.
  • FRONTEND_URL is also allowed.
  • Any origin containing vercel.app is allowed.

6.3 Firmware Variables to Update

  • Set Wi-Fi in main-logic.ino: ssid, password.
  • Set backend URLs (issue, exit verify, people count).
  • Update UID maps in getStudent() and getBook().

6.4 Useful Utility Sketches

  • wifi-connection.ino: confirms network connection and board IP.
  • uid-detection.ino: prints scanned RFID UID to Serial Monitor.

7) URL Config Files

  • urls.example.txt contains placeholders for backend API endpoints.
  • urls-config.txt currently points to deployed backend URL.
  • Keep these in sync with firmware URL constants when changing environments.

8) Deployment Notes (Vercel)

Backend

  • UI/backend/vercel.json routes all requests to server.js with @vercel/node.
  • Set Vercel env vars: MONGO_URI, FRONTEND_URL, TOTAL_BOOKS.

Frontend

  • UI/frontend/vercel.json rewrites /api/* to deployed backend API.
  • VITE_API_BASE_URL is configured as /api.

9) Full Local Run Checklist

  1. Start MongoDB (Atlas or local).
  2. Create backend .env and run backend on port 5000.
  3. Run frontend on port 5173 (Vite default).
  4. Update ESP32 Wi-Fi credentials and URLs.
  5. Upload main-logic.ino to ESP32.
  6. Open Serial Monitor at 115200 baud to verify events and API responses.

Expected: RFID scans create/validate records, home page stats update, and people count syncs from IR sensors.

10) Troubleshooting

  • RFID not detected: verify power (3.3V), SPI pins, SS pins, and RST wiring.
  • CORS blocked: ensure frontend origin is covered by backend CORS allowlist.
  • No DB connection: check MONGO_URI and MongoDB network access rules.
  • Frontend API errors: confirm backend is running at localhost:5000 for local proxy.
  • People count issues: tune IR sensor placement and debounce timing (IR_DEBOUNCE_MS).
